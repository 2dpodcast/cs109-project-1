Predicting Upvotes

Overview


We use the comments and upvotes data within a subreddit to build a predictor on how many votes a comment will receive on a given subreddit. We treat this problem as an example of supervised learning, where we want to perform a regression from feature vectors on text onto an upvote score. To do this, we separated the process into multiple parts.

  1. Upvote Normalization. In order to provide input to our supervised learning models, we normalized our upvotes to the interval [0,1]
  2. Feature Selection. We used different types of models to extract features from text in subreddit comments. These features include a Bag of Words model, N-gram model, and Latent Dirichlet Allocation to discover topics in text.
  3. Dimensionality Reduction. Because the dimensionality of the feature space can be extremely huge (on the order of tens of thousands of features for a small data set), we needed to reduce the dimensionality of the feature space to make the training phase computationally tractable. To do this, we used multiple reduction models, including Kernel Principle Components Analysis and Selecting the K best features
  4. Learning. We used several supervised learning techniques to build a regressor from our feature space X to our normalized upvotes Y. Our supervised learning methods include: Gaussian Naive Bayes, Support Vector Machines, K Nearest Neighbors, and Decision Trees
  5. Putting it all together. We experimented with different combinations of feature selection, dimensionality reduction, and learning models, and we measured the error for each of these combinations through cross validation.
